# AGENTS.md - SecShare Codebase Guide

## Architecture Overview

**SecShare** is an ASP.NET Core API codebase targeting `net10.0` / C# 14. The solution is split into reusable abstractions, API host, business services, NHibernate ORM, FluentMigrator migrations, and xUnit tests.

| Project | Role |
|---|---|
| `SecShare.Api` | ASP.NET Core REST API host |
| `SecShare.Business` | Business services, Autofac modules, middleware, logging, S3 storage client |
| `SecShare.Business.Orm` | NHibernate entities, DAOs, mappings, session factory/provider |
| `SecShare.Migrations` | FluentMigrator console app for PostgreSQL migrations |
| `SecShare.Business.Testing` | Test-only Autofac module, data factories, DB cleanup helpers |
| `SecShare.Tests.Integration.Api` | HTTP/API integration tests via `WebApplicationFactory` |
| `SecShare.Tests.Integration` | Business/DAO integration tests with a direct Autofac container |
| `SecShare.Tests.Unit` | Unit tests |

Shared/cross-cutting libraries:

| Project | Role |
|---|---|
| `Api.Requests.Abstractions`, `Api.Requests.Base` | Custom async request/handler dispatch |
| `AspNetCore.ApiControllers.Abstractions`, `AspNetCore.ApiControllers.Extensions` | Controller base classes and `RequestAsync` helpers |
| `Domain.Abstractions` | Marker interfaces for domain services, entities, queue contexts, domain exceptions |
| `Persistence.Transactions.Behaviors` | DB session/transaction abstractions |

## Request/Handler Pattern (Not MediatR)

API endpoints use the custom `IAsyncRequestHandler<TRequest, TResponse>` / `IAsyncRequestHandler<TRequest>` pattern. Do not introduce MediatR for new endpoints.

Typical layout:

```text
SecShare.Api/Controllers/<Feature>/
  <Feature>Controller.cs
  Actions/
    <Action>Request.cs
    <Action>RequestHandler.cs
    <Action>Response.cs
```

Controllers should stay thin, inherit from `MainApiControllerBase`, and dispatch through `RequestAsync`:

```csharp
[HttpGet]
public Task<IActionResult> Get()
    => this.RequestAsync()
        .For<PingResponse>()
        .With(new PingRequest());
```

Handlers are auto-registered in `SecShare.Api/Di/Autofac/Modules/ApiModule.cs` via `AsClosedTypesOf`. Put endpoint business logic in handlers or domain services, not in controllers.

## Dependency Injection

DI uses Autofac modules. The API host uses `AutofacServiceProviderFactory` and registers:

- `ApiModule` from `SecShare.Api`
- all modules from `SecShare.Business` via `RegisterAssemblyModules(typeof(BusinessAssemblyMarker).Assembly)`

Current business modules:

- `DbModule`: `IDbConnectionFactory`, `IDbSessionProvider`
- `DomainModule`: `IDomainService` implementations from `SecShare.Business`
- `OrmModule`: `IDomainService` implementations from `SecShare.Business.Orm`
- `StorageModule`: `IFileStorageS3Client`

DAO interfaces and domain services should implement `IDomainService` so they are registered automatically. Use `InstancePerDependency` for stateless domain services/DAOs unless there is a clear lifetime reason to do otherwise.

## ORM & Database

The persistence stack is NHibernate + FluentNHibernate against PostgreSQL.

- All entities should extend `AEntity` from `SecShare.Business.Orm/Core/AEntity.cs`.
- `AEntity` provides `Id: Guid`, `CreatedAt`, `UpdatedAt`, `DeletedAt`, `IsDeleted`, and `IsNew`.
- Entity IDs are UUID v7 generated by the app via `GuidV7Generator`; do not rely on DB-generated IDs.
- Entities live in `SecShare.Business.Orm/Entities/`.
- Mappings live in `SecShare.Business.Orm/Mapping/Entities/` and extend `BaseGuidMappings<T>`.
- `SnakeCaseConvention` converts entity/property/reference names to `snake_case`; add explicit mapping overrides only when needed.
- DAOs live under `SecShare.Business.Orm/Dao/` and should use `BaseDao` to access the current NHibernate session.
- Use `DateTime.UtcNow` for new timestamp values.

`CommitPerformerMiddleware` wraps each API request in a transaction: commit/flush on success, rollback/close on exception. Do not manually commit transactions inside handlers unless a task explicitly requires special transaction boundaries.

When adding or changing DAO list/query methods, check for N+1 issues caused by lazy-loaded relationships used by DTO mapping or response construction. Prefer explicit eager fetching or projections where appropriate.

## Migrations

Use FluentMigrator classes in `SecShare.Migrations/Migrations/`. Migrations should inherit from `SecShare.Migrations.Code.MyMigration`.

Run migrations:

```bash
dotnet run --project ./SecShare.Migrations
```

The migration app applies `ConnectionStrings:DefaultConnection`. If `ConnectionStrings:TestConnection` is configured, it applies migrations there too. `ASPNETCORE_CONNECTION_STRING` overrides the default connection string for the migration process.

Use PostgreSQL-compatible `snake_case` names. For date/time columns, prefer `timestamptz` unless the existing schema or task explicitly requires a different type.

## Configuration

Shared configuration is in `Shared/appsettings.json` and is linked into runnable/test projects as `appsettings.json`.

Configuration load order for the API/business code is:

```text
appsettings.json
appsettings.Debug.json (DEBUG only, optional)
appsettings.{ASPNETCORE_ENVIRONMENT}.json (optional)
appsettings.Testing.json (optional)
appsettings.Local.json (optional)
environment variables
```

Important keys:

- `ConnectionStrings:DefaultConnection` - main PostgreSQL connection; defaults to `localhost:5432`, database `secshare`
- `ConnectionStrings:TestConnection` - optional migration target for test DBs
- `Hibernate:IsShowSql` - set to `true` to log SQL via the NHibernate interceptor
- `AWS:S3:BucketName`, `AWS:S3:AccessKey`, `AWS:S3:SecretKey` - required by `FileStorageS3Client`

Do not commit real secrets. Use `appsettings.Local.json`, user secrets, or environment variables for local credentials.

## File Storage

File storage currently uses S3 through `IFileStorageS3Client` / `FileStorageS3Client`.

- `FileStorage.MaxFileSize` is 50 MB.
- Stored object keys are generated under `files/yyyyMMdd/<uuid-v7>.<extension>`.
- Metadata is persisted in the `files` table through `IFilesDao`.
- `DeleteFileAsync` deletes the S3 object and soft-deletes the DB row by setting `DeletedAt`.

When adding storage behavior, keep S3 calls behind `IFileStorageS3Client` or another interface so tests can replace external I/O.

## Testing

Test projects:

| Project | Type | Base class |
|---|---|---|
| `SecShare.Tests.Integration.Api` | HTTP/API integration | `Core/BaseTest : IClassFixture<ApiCustomWebApplicationFactory>` |
| `SecShare.Tests.Integration` | Business/DAO integration | `Core/BaseTest` |
| `SecShare.Tests.Unit` | Unit | none currently |

Useful test helpers:

- `CleanUpDbAsync()` deletes rows from known test tables.
- `FlushDbChanges()` flushes NHibernate session changes.
- `FlushAndRefreshEntity(entity)` flushes and reloads an entity.
- `CreateFormFile()` creates an in-memory `IFormFile`.
- `IDataFactory<TEntity>` implementations live in `SecShare.Business.Testing/Factories/`.

Run checks:

```bash
dotnet build ./SecShare.sln
dotnet test ./SecShare.Tests.Integration.Api
dotnet test ./SecShare.Tests.Integration
dotnet test ./SecShare.Tests.Unit
```

Integration tests require PostgreSQL and migrated schema matching the configured connection string.

## API Responses & Exceptions

`MainApiControllerBase.Success` returns `200 OK` with `JsonCommonResponse` for no-response requests. Response requests return the response object through `OkObjectResult`.

Invalid model state returns `400 Bad Request` with validation errors through `ApiControllerBase.InvalidModelState`.

`IDomainException` exists in `Domain.Abstractions`, but this repository currently has no domain-exception HTTP filter/middleware like TimeVic. Do not assume domain exceptions automatically become structured HTTP 400 responses until such handling is added.

## Code Style & Conventions

- Write comments and implementation notes in English.
- Do not create git commits unless the user explicitly asks.
- Keep controllers thin; put logic in handlers, DAOs, or domain services.
- Boolean variables/properties should start with `Is`, `Has`, `Can`, or another clear boolean prefix.
- Interface names must start with `I`.
- Place nested classes/interfaces/records near the top of the containing type unless local style says otherwise.
- Avoid noisy comments that only restate a trivial action.
- If a change fixes a subtle bug or a non-obvious edge case, add a concise comment explaining the issue being guarded against.
- Preserve nullable reference types and implicit usings style used by existing projects.
- Prefer project-local abstractions over direct framework/external dependencies in business code when that improves testability.
